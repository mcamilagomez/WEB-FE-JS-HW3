What is JavaScript

JavaScript is a scripting or programming language that allows you to implement complex features on web pages — every time a web page does more than just sit there and display static information for you to look at — displaying timely content updates, interactive maps, animated 2D/3D graphics, scrolling video jukeboxes, etc. — you can bet that JavaScript is probably involved.



What JavaScript can do:


  • Client-Side Scripting: JavaScript runs in the browser, allowing for dynamic content updates, form validation, animations, and user interactions without needing to reload the page.
  • Event Handling: It can respond to user actions like clicks, key presses, and mouse movements, enabling interactive web applications.
  • APIs and Libraries: JavaScript supports numerous libraries and frameworks (such as React, Angular, and Vue.js) that simplify development and provide robust tools for building complex applications.
  • Cross-Platform Development: JavaScript is used in frameworks like Electron and React Native to build cross-platform desktop and mobile applications.

JavaScript Async

JavaScript async is a keyword used to define asynchronous functions, allowing you to write code that handles non-blocking operations. This means tasks like API calls or file reading can run in the background while other lines of code continue executing. When a function is marked with async, it always returns a promise. If the function returns a value, it automatically wraps it in a resolved promise. If it throws an exception, the promise will be rejected.

What is an asynchronous function?

An asynchronous function in JavaScript is a function designed to handle operations that don’t complete immediately, such as reading files, making network requests, or executing tasks that may take some time to finish.


Basic Syntax

                
                    async function exampleAsync() {
                        return "Hello, world!";
                    }
                      
                    exampleAsync().then(message => console.log(message)); // Prints "Hello, world!"

CallBack

A callback in JavaScript is a function that is passed as an argument to another function and is executed inside that function when an event occurs or after a task is completed. Callbacks allow code to be executed asynchronously or after an operation, such as a network request or a timer, is finished.

Basic Example

                
                    function greet(name, callback) {
                        console.log("Hello, " + name);
                        callback();
                    }
                      
                    function goodbye() {
                        console.log("Goodbye!");
                    }
                      
                    greet("María", goodbye);

The greet function takes two arguments: name and callback.
Inside greet, the callback (in this case, the goodbye function) is executed.

Output

                
                    Hello, María
                    Goodbye!

Callbacks in Asynchronous Functions

Callbacks are commonly used in asynchronous operations like HTTP requests or timers, allowing the program to continue running without blocking other operations.

                
                    setTimeout(function() {
                        console.log("This runs after 2 seconds");
                    }, 2000);

setTimeout receives a callback (an anonymous function) that will run after 2 seconds.
The code continues executing while waiting for the timer to finish.

Promise

A promise in JavaScript is an object that represents the eventual completion (or failure) of an asynchronous operation. Promises allow you to handle tasks that take time to complete, such as HTTP requests, file access, or timers, without blocking the execution of the code.

States of a Promise



A promise can be in one of three states:
  • Pending: The asynchronous operation has not yet completed or failed.
  • Fulfilled: The operation completed successfully, and a value is produced.
  • Rejected: The operation failed, and an error or failure reason is provided.

Basic Syntax

                
                    let promise = new Promise((resolve, reject) => {
                        // Perform the asynchronous operation here
                        let success = true; // Simulates the success of the operation 
                      
                        if (success) {
                          resolve("Operation successful");  // Calls resolve if everything went well 
                        } else {
                          reject("Operation failed");  // Calls reject if something went wrong
                        }
                    });

Handling Promises



To handle the result of a promise, you use the .then() and .catch() methods:
  • .then():Executes when the promise is fulfilled successfully.
  • .catch():Executes when the promise is rejected.

Example:

                
                    let promise = new Promise((resolve, reject) => {
                        setTimeout(() => {
                          resolve("Operation completed");
                        }, 2000);
                    });
                      
                    promise
                        .then(result => {
                            console.log(result);  // "Operation completed" after 2 seconds
                        })
                        .catch(error => {
                            console.error(error);  // In case of error 
                        });

Advantages of Promises

Avoid callback hell

Promises help avoid excessive nesting of callback functions, improving code readability.

Clear error handling

With .catch(), you can handle errors consistently without needing multiple try...catch blocks.

Chaining operations

Promises allow you to chain multiple asynchronous operations in a clean manner.

Await

Await is a keyword in JavaScript used within functions declared with async to pause the execution of the function until a promise is resolved or rejected. By using await, you can work with asynchronous results as if they were synchronous, making your code easier to read and write.

Basic Example

                
                    async function example() {
                        let result = await someAsyncFunction();
                        console.log(result);
                    }
                      
                    example();
In this example:
  1. someAsyncFunction(): Is a function that returns a promise.
  2. await: pauses the execution of the example function until the promise returned by someAsyncFunction() is resolved.
  3. Once the promise resolves, the resulting value is assigned to result, and the code continues executing.

Eventloop

The event loop is a fundamental mechanism in JavaScript and other programming languages for managing asynchronous code execution. Its main purpose is to allow a single thread to handle multiple tasks without blocking the execution flow of the program.

How it works

  1. Call Stack: This is the data structure that keeps track of the functions that are currently being executed. When a function is called, it is added to the stack, and when it completes, it is removed.
  2. Task Queue: This is where asynchronous tasks are placed once they are complete and ready to be executed. These tasks are processed after the call stack is empty.
  3. Event Loop: The event loop monitors the call stack and the task queue. Its job is to ensure that when the call stack is empty, tasks from the task queue are moved to the call stack for execution.

General process


  • Code execution happens on the call stack.
  • When an asynchronous task (like a promise or a timer) is encountered, it is sent to the task queue.
  • The event loop continues executing code on the call stack. Once the stack is empty, the event loop moves tasks from the task queue to the call stack for execution.

Example

                
                    console.log('Start');

                    setTimeout(() => {
                        console.log('Timeout');
                    }, 0);

                    console.log('End');

Output

                
                    Start
                    End
                    Timeout

Even though the timer is set to 0 milliseconds, console.log('Timeout') executes after the other two console.log statements because setTimeout adds the function to the task queue, and the event loop processes it after the synchronous code in the call stack has completed.
